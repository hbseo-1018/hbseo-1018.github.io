Socket.io 기반 사용자별 실시간 알림 기능을 구현

상담 담당자 배정, 결재 요청, 회원가입 승인 요청 등 즉시 확인이 필요한 이벤트가 발생하면 알림을 먼저 DB에 저장하고, 저장된 알림 데이터를 사용자 ID 기반 room으로 emit하여 대상 사용자에게만 전달. Redis Adapter를 적용해 서버 확장 시에도 Socket.io 이벤트가 인스턴스 간 공유될 수 있도록 구성. 클라이언트에서는 WebSocket 연결 후 join(userId) 이벤트로 개인 room에 입장하고 notification 이벤트 수신 시 알림 목록, 미확인 카운트, toast UI를 즉시 갱신하도록 처리.
